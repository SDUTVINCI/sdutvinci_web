# Vinci 机器人队成员资料填写与提交教程

这份教程面向不了解 Markdown（`.md`）和编程的成员。你不需要安装开发环境，也不需要会写代码，只需要用文本编辑器修改模板并提交两个文件。

## 最终需要提交什么

每位成员需要提交以下两个文件：

| 文件 | 命名示例 | 说明 |
| --- | --- | --- |
| 成员资料 | `张三.md` | 文件名必须是本人姓名，后缀必须是 `.md` |
| 个人照片 | `张三.jpg` | 照片文件名必须是本人姓名，并与资料中的图片路径一致 |

例如，成员“张三”最终应准备：

```text
张三.md
张三.jpg
```

不要只提交照片，也不要把资料保存成 Word 文档、PDF 或 `张三.md.txt`。

## 第一步：复制并重命名模板

1. 找到本文件夹中的 [`成员资料模板.md`](./成员资料模板.md)。
2. 复制一份模板，不要直接修改原始模板。
3. 把复制出的文件重命名为“你的姓名 + `.md`”。

例如：

```text
成员资料模板.md → 张三.md
```

### Windows 看不到文件后缀怎么办

打开任意文件夹，在顶部选择：

```text
查看 → 显示 → 文件扩展名
```

确认文件完整名称是 `姓名.md`，而不是 `姓名.md.txt`。

## 第二步：用普通文本编辑器打开

Windows 用户可以：

1. 右键点击 `.md` 文件。
2. 选择“打开方式”。
3. 选择“记事本”。

macOS 用户可以用“文本编辑”，但要先选择“格式 → 制作纯文本”。

不要使用 Word 或 WPS 直接保存，因为它们通常会把文件变成 `.docx`，网站无法读取。

保存时如果可以选择编码，请选择 `UTF-8`。新版 Windows 记事本默认就是 UTF-8。

## 第三步：填写开头的基本资料

模板开头由两行 `---` 包围。这一段是网站读取成员信息的区域：

```yaml
---
name: 你的姓名
image: /images/member_photo/你的姓名.jpg
role: 控制组成员
type: 控制组
time: "26"
advisor: null
grade: "2024"
affiliation: 机械工程学院
links: null
---
```

不要删除开头和结尾的 `---`，不要修改 `name`、`image` 等英文名称，只修改冒号后面的内容。

### 每个字段怎么填写

| 字段 | 填写方法 |
| --- | --- |
| `name` | 本人真实姓名，必须与 `.md` 文件名和照片文件名一致 |
| `image` | 固定以 `/images/member_photo/` 开头，后面填写照片完整文件名 |
| `role` | 在队职责，可以填写多个，用中文逗号隔开，例如 `控制组组长，副队长` |
| `type` | 只能选择：`团队负责人`、`机械组`、`控制组`、`电路组`、`算法组`、`运营组` 或 `指导老师` |
| `time` | 作为队员参加的赛季年份后两位，例如 2026 年填写 `"26"`；多个赛季可写 `"24,25,26"` |
| `advisor` | 后续担任顾问的赛季，例如 `"25,26"`；没有则保持 `null` |
| `grade` | 入学年级，填写四位年份，例如 `"2024"` |
| `affiliation` | 所属学院全称，例如 `机械工程学院` |
| `links` | 没有个人主页等公开链接时保持 `null` |

### `role` 和 `type` 有什么区别

- `role` 是详细职责，可以写多个，例如“控制组组长，队长，机电创新学会会长”。
- `type` 是网站用来分类的主要身份，只能保留一个。
- 队长、副队长等主要负责人使用 `团队负责人`；普通成员和组长按自己的技术组填写。

### 有个人链接时怎么填写

如果没有公开链接，保持：

```yaml
links: null
```

如果有链接，把这一行替换成下面的格式。每个网址前面必须保留两个空格：

```yaml
links:
  github: https://github.com/你的账号
  home-page: https://你的个人主页
  bilibili: https://space.bilibili.com/你的UID
```

没有的项目直接删除，不要填写“无”。

## 第四步：填写个人介绍

第二个 `---` 下面是个人介绍。这里可以直接写普通中文，不会 Markdown 语法也没有关系。

推荐写以下内容：

```markdown
一句话自我介绍或想对队友说的话。

- 负责过的机器人或项目
- 参加过的比赛与获得的奖项
- 擅长的技术方向
- 愿意公开的联系方式
```

以 `-` 开头的行会显示为列表。个人介绍也可以留空。

网站是公开访问的。手机号、QQ、微信、私人邮箱等信息只有在你愿意公开时才填写，不要提交身份证号、家庭住址等敏感信息。

## 第五步：准备个人照片

照片建议满足以下要求：

- 优先使用清晰的单人正面照或半身照。
- 建议使用 JPG 或 PNG 格式，不要直接提交 HEIC 格式。
- 建议文件小于 5 MB，避免过大的原图加载缓慢。
- 文件名使用本人姓名，例如 `张三.jpg`。
- 照片后缀必须与 `image` 字段完全一致。

例如照片是 `张三.png`，资料中也必须写：

```yaml
image: /images/member_photo/张三.png
```

注意 `.jpg`、`.jpeg` 和 `.png` 是不同的后缀，不能混用。

## 提交前检查清单

提交前逐项检查：

- [ ] `.md` 文件名是本人姓名，例如 `张三.md`
- [ ] 文件没有变成 `.md.txt`、`.docx` 或 `.pdf`
- [ ] 照片文件名是本人姓名，例如 `张三.jpg`
- [ ] `name` 与两个文件的姓名完全一致
- [ ] `image` 中的文件名和照片名称、后缀完全一致
- [ ] 两行 `---` 都保留着
- [ ] `type` 使用了规定的分类名称
- [ ] 没有顾问赛季或个人链接时填写 `null`

## 常见问题

### 文件变成了 `张三.md.txt`

打开 Windows 的“文件扩展名”显示，将完整文件名改回 `张三.md`。系统询问是否更改扩展名时选择“是”。

### 中文打开后变成乱码

用记事本重新选择“另存为”，编码选择 `UTF-8`，文件类型选择“所有文件”，再保存为 `姓名.md`。

### 照片在网站上显示不出来

检查照片名称和 `image` 字段是否逐字一致，尤其检查 `.jpg`、`.jpeg`、`.png` 后缀。

### 收集链接打不开

确认地址完整包含 `:5666`。可以更换浏览器或网络后重试；仍然无法打开时联系负责人。
